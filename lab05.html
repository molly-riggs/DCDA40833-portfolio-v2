Vibe Coding Reflection

For the vibe coding lab, I chose to do experiment A that involved adding an element to CSS, and experiment C which involved debugging code. Adding the element of dark mode toggle was something that interested me, since I know some people prefer to have the option when it comes to how they view websites. Debugging code was also an easy choice for me as well, since my home page was messed up from a previous lab. I knew that there was an issue in the code I previously wrote, so I thought this would be a great use of time to experiment with AI and see how I could fix it. For experiment A, I decided to have AI help me add a feature at the top of my website that allowed users to switch between dark and light mode when viewing the website. I wanted to make this feature visually appealing, so I asked AI to make the dark mode option a moon emoji, and the light mode option a sun emoji. That way, users could click on the emoji and switch between dark mode and light mode as they pleased. I chose to use the chatbot Claude Opus 4.6, and was very specific in how I prompted it to complete the task. I asked it to explain each change made to complete the dark mode toggle, and it walked me through a step by step breakdown of what code needed to be added and why. This was super helpful to my learning, and a good reference point to look back on if I need to implement this in other projects. For experiment C, I chose to debug my home page. I had really messed up the home section of the website. The AI lab template for short responses was showing, the navigation bar showed twice, and the about me section wasn’t showing at all. Since the AI lab was popping up on my home page, I knew that I must have duplicated code from lab two, and put it into the index file and the lab 2 file, instead of just the lab 2 file. I found the duplicated lines and deleted them. I was confused as to where my about me section had gone, and AI helped me figure that out. Since I had duplicated the AI section, VS code saw the second and thought the page was done. Because of this, it completely cropped out my about me section. So I had the AI bot help me debug the code and fix this error. Having AI pinpoint the errors in the code and help me solve them really streamlined the coding process, and simplified tasks significantly. It was also helpful that the chatbot would give specific lines that the code errors were in, which made it easier to identify the problems without scrolling aimlessly to find them. I didn’t run into any issues of AI being problematic this time which was nice. It didn’t break any code that I noticed, it helped me not hurt me. I believe that vibe coding can honestly both help and hurt learning. For example, when I was having AI help me implement the dark mode toggle, it walked me through the steps I needed to take and explained why they were necessary and what they did. I think this is a great way to learn, and is a good example of AI aiding learning. However when the agent mode is used on chatbots, it can be easy to have AI do all of the work, which hurts learning. If AI is doing all of the coding and you aren't bothering to ask for explanations or instructions on what is happening, that hurts your learning in the long run. I believe it is most helpful when it comes to debugging, so I want to try to mainly use AI when I have errors that I can’t figure out on my own.
